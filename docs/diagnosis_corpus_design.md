# 结构化诊断样本库说明

这份样本库是为了让 `DiagnosisEngine` 在开源仓库中也能有一套可运行、可演示、可讲解的基础数据。

## 设计目标

- 覆盖 PCB / FPCB 常见工序，而不是只聚焦单一焊接场景
- 每个工序至少有一组“异常 - 原因 - 条件 - 方案”关系
- 让 Web 表单可以按工序动态生成字段，避免无关输入长期常驻
- 作为开源样本使用，不声称等同真实工厂知识库

## 当前覆盖工序

- PI膜下料
- 导电浆料印刷
- 烘烤固化
- 曝光对位
- 显影/蚀刻
- 清洗干燥
- 双面胶/绝缘层贴合
- 冲孔冲型
- 预焊接
- 焊线
- 模压定型
- 电阻/耐压/外观检测

## 当前样本结构

- `processes`: 工序主数据
- `symptoms`: 异常现象字典
- `causes`: 根因画像，带工序范围与条件字段
- `solutions`: 针对根因的建议动作
- `cases`: 历史案例样本
- `rules`: 显式规则，用于快速命中
- `process_observation_templates.json`: Web 诊断页的工序输入模板

## 使用边界

- 这是一套“为了让结构化诊断链跑通并可演示”的样本数据
- 字段和阈值强调可理解、可展示，不等价于真实量产参数标准
- 如果后续接入真实现场知识，优先替换 `causes / rules / cases / templates` 四部分
